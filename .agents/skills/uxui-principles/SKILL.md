# 1. Purpose
Ensure the product is intuitive, accessible, and contextually logical for the user.

# 2. When to Trigger
Designing complex forms, multi-step wizards, user onboarding, navigation.

# 3. When NOT to Trigger
Visual beauty tweaks, algorithmic logic, broad startup/product ideation, or frontend code implementation.

# 4. Scope Boundaries
Usability, behavioral flow, interaction friction, information architecture. MUST NOT become generic design taste, visual aesthetics, or generic product strategy.

# 5. Operational Phase
OPERATIONAL_PHASE: PLANNING

# 6. Tool / MCP Discipline
**ALLOWED_TOOLS**: read_file
**FORBIDDEN_TOOLS**: None
**REASONING_ONLY_OK**: Wireframe logic generation.

# 7. Required Output Format
User Flow definitions and structural wireframe requirements.

# 8. Failure Mode / Risk Awareness
Creating beautiful elements that leave users confused on where to click.
